# AI Package

> 결정론적 바이너리 패턴 탐색과 선택적 AI 사전 후보 선정을 결합한 실험적 무손실 압축 포맷입니다.

[English](README.md)

AI Package는 압축 데이터, 반복 바이트 사전, 파일 메타데이터, SHA-256 체크섬을 하나의 `.aip` 파일에 저장합니다. 여러 파일도 ZIP이나 다른 압축 포맷으로 감싸지 않고 자체 번들 형식으로 처리합니다.

## 시작 가이드

### 요구 사항

- Python 3.10 이상
- 선택 사항: 로컬 LLM 후보 선정을 위한 [Ollama](https://ollama.com/)

### 설치 없이 실행

```bash
git clone <repository-url>
cd aip
python3 -m aip.cli serve --open
```

브라우저 UI는 `http://127.0.0.1:8765`에서 실행됩니다.

### CLI

```bash
# 내장 알고리즘
aip compress sample.bin

# 로컬 Ollama 모델
aip compress sample.bin --ai --model qwen2.5-coder:7b

# 복원 및 SHA-256 검증
aip decompress sample.bin.aip restored.bin
```

웹 UI에서는 여러 파일, 외부 AI API, 드래그 앤 드롭, 한글/영어 전환, 업로드 진행률, 압축 해제 후 원본 파일 비교 기능을 추가로 제공합니다.

## 압축 방식

실제 인코딩과 디코딩은 항상 결정론적인 AIP 코드가 수행합니다. AI는 압축 페이로드를 직접 만들지 않으며, 알고리즘이 먼저 탐색하고 검증한 바이트 패턴 후보의 ID만 선택할 수 있습니다.

전체 압축 과정은 다음과 같습니다.

1. 입력 파일을 원시 바이트로 읽습니다. 여러 파일은 파일명, 크기, 내용을 포함하는 AIP 자체 번들로 먼저 결합합니다.
2. 여러 고정 길이로 반복 바이트 구간을 찾고 각 구간의 출현 횟수를 계산합니다.
3. 사전 저장 공간과 참조 토큰 비용을 제외한 실제 예상 절감량을 계산합니다.
4. 내장 알고리즘, Ollama 또는 외부 AI API로 사전 후보를 선택합니다.
5. 전체 입력을 처음부터 끝까지 읽으며 가장 작은 표현이 가능한 토큰으로 변환합니다.
6. 헤더, 사전, 토큰 스트림, 원본 크기, SHA-256 체크섬을 하나의 `.aip` 파일에 기록합니다.

### 토큰 인코딩

압축 페이로드는 네 가지 AIP 자체 토큰을 사용합니다.

- **리터럴 블록**은 참조하는 것보다 그대로 저장하는 것이 효율적인 바이트를 보관합니다.
- **사전 참조**는 파일 전체에서 반복되는 바이트 구간을 사전 인덱스로 대체합니다.
- **반복 길이 토큰**은 동일한 바이트 하나와 반복 횟수만 저장합니다.
- **역참조**는 앞에서 이미 나타난 바이트 구간의 거리와 길이만 저장합니다.

인코더는 사용할 수 있는 일치 항목들을 비교하고 일반적으로 가장 긴 유효 표현을 선택합니다. 디코더는 토큰을 순서대로 처리하여 원본 바이트를 복원한 뒤 SHA-256 체크섬을 검증합니다.

#### 인코딩 예시

다음과 같은 단순화된 입력이 있다고 가정합니다.

```text
HEADER|abcabcabcabc|HEADER|HEADER|
```

예시 사전은 다음과 같이 구성될 수 있습니다.

```text
0 = "HEADER|"
```

개념적으로 페이로드는 다음과 같이 표현할 수 있습니다.

```text
DICT(0)
LITERAL("abc")
BACK_REFERENCE(distance=3, length=9)
DICT(0)
DICT(0)
```

`abcabcabcabc`는 `abc` 한 번을 리터럴로 저장한 뒤 앞의 3바이트 패턴을 반복하는 역참조로 표현됩니다. 위 표기는 동작을 설명하기 위한 것이며 실제 `.aip` 파일에는 작은 바이너리 연산 코드와 가변 길이 정수가 저장됩니다.

### 내장 알고리즘

AI나 네트워크 요청을 사용하지 않습니다.

후보 패턴을 예상 바이트 절감량 순서로 정렬합니다. 너무 짧거나, 이득이 없거나, 중복되거나, 더 강한 항목에 완전히 포함되는 패턴은 제거합니다. 선택된 사전은 토큰 변환 과정에서 반복 길이 및 역참조 탐색과 함께 사용됩니다.

예시:

```text
입력 후보:
ID 0: "customer_id"  길이=11  출현=120  예상 절감=945
ID 1: "customer"     길이=8   출현=120  예상 절감=590
ID 2: "xyz"          길이=3   출현=2    예상 절감=-4

선택:
ID 0
```

ID 1은 더 강한 ID 0에 포함되므로 중복이고, ID 2는 사전에 저장하는 비용이 절감량보다 큽니다.

### Ollama 로컬 LLM

모든 후보는 먼저 결정론적 탐색기가 찾습니다. 로컬 모델에는 후보 ID, 길이, 출현 횟수, 예상 절감량, Base64 후보 바이트가 전달됩니다. 모델은 후보의 겹침과 중복 관계를 분석한 뒤 다음 형식의 ID만 반환합니다.

```json
{"selected_ids":[1,2,3]}
```

알 수 없는 ID는 폐기하며, Ollama가 실행되지 않았거나 응답이 잘못되면 내장 알고리즘으로 자동 전환합니다. Ollama 연결 주소는 `localhost`, `127.0.0.1`, `::1`만 허용합니다.

예시:

```text
AIP 탐색기 -> 후보 [0, 1, 2, 3, 4]
Ollama     -> {"selected_ids":[0,3]}
AIP 인코더 -> ID 0과 3을 검증하고 사전을 만든 뒤 파일을 인코딩
```

모델이 `999`처럼 존재하지 않는 ID를 반환해도 인코더는 이를 무시합니다.

### 외부 AI API

웹 UI에서 HTTP 메서드, 채팅 API URL, 헤더, 본문 템플릿을 설정할 수 있습니다. 검증된 후보 선택 요청이 들어갈 위치에 `{{data}}`를 사용합니다.

```json
{
  "model": "your-model",
  "stream": false,
  "messages": [
    {
      "role": "user",
      "content": "{{data}}"
    }
  ]
}
```

응답은 `selected_ids`를 직접 포함하거나, 채팅 응답 문자열 또는 `json` 코드 블록 안에 포함할 수 있습니다. 응답이 잘못되면 내장 알고리즘으로 자동 전환합니다.

응답 형식 예시:

```json
{"selected_ids":[0,3,7]}
```

````text
```json
{"selected_ids":[0,3,7]}
```
````

```json
{
  "choices": [
    {
      "message": {
        "content": "{\"selected_ids\":[0,3,7]}"
      }
    }
  ]
}
```

`{{data}}`에는 후보 선택 지시문과 후보 데이터셋이 포함됩니다. 외부 API가 임의의 압축 바이트를 만드는 구조가 아니며, 반환된 모든 ID는 사용 전에 검증됩니다.

### 파일 묶음과 무결성

- 여러 파일은 ZIP이 아닌 AIP 자체 번들 형식으로 저장됩니다.
- 압축 해제 시 원본 파일들을 직접 복원합니다.
- 선택적으로 원본 파일과 SHA-256을 비교할 수 있습니다.
- 새 압축 파일은 ZIP이나 DEFLATE를 사용하지 않습니다.
- 초기 실험 버전에서 생성된 DEFLATE 플래그 AIP 파일은 압축 해제 호환성을 유지합니다.

여러 파일 압축 예시:

```text
photos/logo.png
docs/readme.txt
data/sample.bin
        │
        ▼
AIP 자체 파일 번들
        │
        ▼
사전 + 토큰 스트림 + SHA-256
        │
        ▼
project-files.aip
```
